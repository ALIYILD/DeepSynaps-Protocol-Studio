## Product-ingest export

The compact ingest export is generated by:

```bash
python3 services/evidence-pipeline/export_neuromodulation_ingest_csv.py
```

It keeps the product-relevant columns only:

- identity: `paper_key`, `doi`, `pmid`, `pmcid`
- bibliographic: `title`, `authors`, `journal`, `year`, `pub_type`
- content: `abstract`, `research_summary`
- evidence tagging: `study_type_normalized`, `evidence_tier`, `paper_confidence_score`, `priority_score`
- neuromodulation tagging: `primary_modality`, `canonical_modalities`, `indication_tags`, `condition_mentions_top`, `target_tags`
- safety/outcomes: `safety_signal_tags`, `contraindication_signal_tags`, `outcome_snippet_count`, `outcome_categories`, `real_world_evidence_flag`
- linkage: `record_url`, `trial_top_nct_ids`, `fda_top_clearances`

## Priority review export

The clinician-review slice is generated by:

```bash
python3 services/evidence-pipeline/export_neuromodulation_priority_review_csv.py --limit 5000
```

It ranks rows by:

- `priority_score`
- `paper_confidence_score`
- `cited_by_count`
- `year`

and keeps a 26-column review-friendly subset with title, abstract, DOI/PMID,
modality, indication, evidence tier, citations, trial/FDA linkage, and summary.

## Refresh commands

From repo root:

```bash
python3 services/evidence-pipeline/import_neuromodulation_bundle.py --stage
python3 services/evidence-pipeline/import_neuromodulation_bundle.py --import-db
```

If local Python has the required workbook deps installed, rebuild from the staged bundle with:

```bash
python3 services/evidence-pipeline/build_desktop_neuromodulation_database.py \
  --bundle-root data/research/neuromodulation \
  --journal-backfill-limit 0
```

To import the staged bundle into a dedicated SQLite file without touching the
shared `evidence.db`, run:

```bash
python3 services/evidence-pipeline/import_neuromodulation_bundle.py \
  --import-db \
  --db services/evidence-pipeline/neuromodulation_evidence_2026-04-29_v2.db
```

The importer now runs without a single giant transaction, so row counts become
visible while the import is still in progress.
