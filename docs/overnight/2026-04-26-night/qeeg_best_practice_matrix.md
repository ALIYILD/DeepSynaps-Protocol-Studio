# qEEG Best-Practice Matrix — 2026-04-26 Night Shift

Comparison: current implementation vs MNE-Python best practice + community standards (PREP, FOOOF, Brainstorm, ICLabel, AutoReject, ASR).

| Area | Current | Best Practice | Gap | Tonight? | Rationale |
|---|---|---|---|---|---|
| File I/O | MNE-Python readers for EDF/EDF+/BDF/BV/EEGLAB/FIF; canonical 10-20 synonym map; `make_standard_montage('standard_1020')` with `on_missing='warn'` | MNE-BIDS native loaders + montage from BIDS metadata when present; explicit unit validation; reject on bad signal-units (V vs µV) | Pipeline trusts MNE's unit handling; no explicit `with_units` check; upload allowlist excludes `.vhdr/.set/.fif` even though pipeline supports them | N | Format support is the right priority for night shift; allowlist mismatch is a 1-line tweak but lives in upload route which the QA stream will revisit. |
| Bad-channel detection | PyPREP robust avg ref (preferred) → silent fallback to plain avg ref when PyPREP missing | PREP pipeline (Bigdely-Shamlo 2015): line-noise removal + RANSAC + correlation + deviation criteria + iterative referencing. | Bandpass / Notch / Resample | FIR firwin zero-phase 1–45 Hz; configurable notch (50/60); resample 250 Hz post-filter | MNE recommends firwin+zero-phase, 1 Hz HP for ICA, notch at line + harmonics if visible, downsample post-filter to avoid aliasing | Notch covers fundamental only, no harmonic detection (e.g., 100/150 Hz); skip-by-annotation already correct | N | Harmonic-aware notch is nice-to-have; current 1–45 Hz cutoff makes harmonics moot. |
| Epoching | 2 s, 50 % overlap, edge discard 10 s/5 s | MNE resting-state convention: 2-4 s, 50 %, edge discard ≥5 s; 30+ epochs target | Already in spec | N | Solid. |
| ICA | picard (fallback infomax); n_comp = min(n_eeg-1, 30); fit on 1 Hz HP copy | MNE/ICLabel docs: picard or extended-infomax; fit on 1 Hz HP; ICLabel proba threshold 0.7-0.9 | Threshold matches | N | Solid. |

| Epoch rejection | AutoReject local | AutoReject local is the reference standard | None | N | Solid. |
| ASR (Artifact Subspace Reconstruction) | Not implemented | EEGLAB clean_rawdata / mne-asr provide subspace cleaning when ICA fails for highly-contaminated recordings | Not used; would need optional dep | N | Out of scope — would require new dependency; PyPREP+ICA+AutoReject covers most clinical cases. Noted as future work. |

| FOOOF / SpecParam | Per-channel slope, offset, R²; PAF from peak_params; argmax fallback | SpecParam (Donoghue 2020): aperiodic + periodic decomposition with `peak_width_limits=(2,12)`, knee mode for low-frequency, R² >0.9 quality threshold | No knee-mode auto-detect (delta-band fits may be poor); no R² QC flag surfaced per-channel | **Y** | Key paper convention: gate per-channel PAF/slope reliability by R². Add channel-level confidence flag. |
| Asymmetry | F3/F4 and F7/F8 frontal alpha (Davidson approach/withdrawal) | Coan & Allen (2004) recommend F3/F4 + F7/F8 (covered) and posterior P3/P4 for relaxation contrast; recommend median across pairs | Posterior pair missing; OK for night shift | N | Frontal pair is the validated clinical indicator. |
| Connectivity | wPLI + coherence (multitaper); zero-fill on failure | MNE-Connectivity recommends wPLI as gold-standard for volume-conduction safety; surrogate stats (e.g., 100 phase-randomised surrogates) for significance threshold | No surrogate / null-distribution; raw wPLI is hard to interpret without baseline | N | Surrogates ×100 too heavy for night-shift add. | Graph metrics | Top-20 % edge threshold, clustering, CPL, small-worldness from configuration-model surrogates ×10 | Network neuroscience consensus (Bullmore & Sporns 2009; Rubinov & Sporns 2010): proportional thresholding; report range over thresholds (e.g., 10/15/20/25 %); use Brain Connectivity Toolbox conventions | Single threshold; no robustness sweep | N | Single-threshold is fine for v1; sweep is roadmap. |
| Source localisation | eLORETA on fsaverage; quality-gated (≥19 ch, ≥20 epochs, <30 % bad) | MNE recommends eLORETA or sLORETA for resting state; subject-MRI preferred; fsaverage acceptable with explicit limitations stamp | Limitations not surfaced per-channel/ROI; no sLORETA fallback | N | eLORETA is the right primary choice; gating is correct. |
| Normative z-scores | Norm DB lookup with `flagged` list at \|z\|>1.96 / >2.58 | Brainstorm + LORETA Key conventions; report z + SE / CI; age/sex stratified | No SE/CI; flagged list shape is good | N | Norm DB internals out of scope. |

| QC flags | Coarse list inside clinical_summary | Brainstorm-style explicit QC envelope with code, severity, message, affected_channels, recommendation | Codes exist; affected_channels + recommendation missing; not surfaced as a top-level pipeline output | **Y** | Frontend needs structured array to render per-finding badges. |

## Tonight's shipping list (Y rows above)

1. Fallback bad-channel detector (correlation + deviation criteria) when PyPREP unavailable.
2. Per-feature confidence (`confidence` dict) on spectral / aperiodic / asymmetry features (n_epochs / R² / channel_validity-based).
3. FOOOF channel-level R²-based reliability flag + structured channel-validity output.
4. Top-level pipeline output schema additions: `qc_flags`, `method_provenance`, `confidence`, `limitations` arrays — promoted out of `clinical_summary` so all consumers can read them without descending.
5. Structured stage-error envelope (code, message, severity, recoverable).
6. Evidence-linkage hook in clinical_summary findings — calls evidence layer if reachable, marks `evidence_pending` otherwise. Never fabricates citations.
7. Frontend: render `qc_flags` array, observed-vs-inferred sectioning, per-finding evidence chips with "evidence pending" fallback, no fake buttons.
8. Tests covering each of the above.
