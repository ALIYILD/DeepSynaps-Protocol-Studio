# Narrative RAG (qEEG report Discussion)

## Goal
Generate a clinician-facing **Discussion** section for the DeepSynaps qEEG report that is:
- **Grounded** in the DeepSynaps 87k-paper literature DB (via MedRAG retrieval)
- **De-identified** (no patient name/MRN or other direct identifiers)
- **Citation-strict** (every sentence must contain an in-scope citation id)
- **Regulatorily conservative** (decision support / research-wellness use only)

Scope is intentionally narrow: this module only populates the report’s Discussion/References, and it does not make diagnoses or treatment decisions.

## Architecture (high level)
Pipeline:
1) **Findings extraction** (`narrative/findings.py`)
   - Reads `PipelineResult.zscores["flagged"]` rows and converts them into `Finding` objects.
   - Applies thresholds:
     - \(|z| \ge 2.0\) → `severity="significant"`
     - \(1.5 \le |z| < 2.0\) → `severity="borderline"`
   - Sets direction from z sign: `elevated` / `reduced` / `normal`.

2) **Evidence retrieval** (`narrative/retrieve.py`)
   - Uses existing MedRAG (`deepsynaps_qeeg/ai/medrag.py`) to retrieve papers from the DeepSynaps literature DB.
   - Retrieval is blended:
     - **60% graph-style pass**
     - **40% vector-style pass**
   - Returns `Citation` objects (pmid/doi/title/year/url) and preserves a stable `citation_id` (`C1`, `C2`, …).

3) **Narrative composition** (`narrative/compose.py`)
   - Uses a **provider abstraction** (`NarrativeProvider`) so no vendor SDK is hardcoded.
   - Prompt requirements enforce:
     - structured sections
     - no PHI
     - no citations outside the retrieved set
     - **every sentence** has `[...]` citation markers
   - Default provider is `MockNarrativeProvider` (deterministic, offline-safe).

4) **Safety + consistency** (`narrative/safety.py`)
   - Deterministic “consistency checker” enforces:
     - every sentence contains at least one `[C#]`
     - every cited id exists in the allowed retrieved set
   - If generation fails, the system:
     - reprompts up to **2** times (if a real provider is supplied)
     - otherwise emits a **deterministic fallback narrative** that is guaranteed to pass the checker

## PHI policy
Only de-identified info may be passed to any LLM provider:
- Allowed: age/sex and generic recording metadata (if provided)
- Disallowed: name, MRN, DOB, addresses, contact info, free-text notes with identifiers

The narrative module does not accept or transmit direct identifiers; callers must keep any additional metadata de-identified.

## Report integration
Integration is constrained to `deepsynaps_qeeg/report/generate.py`:
- Adds a **Discussion** section populated by narrative Markdown converted to HTML.
- Adds a **References** section with DOI links (or PubMed links).
- Adds a footer line on pages containing narrative:
  - “Generated by AI under clinician supervision”

## Testing
Unit tests in `tests/test_narrative.py` cover:
- findings extraction produces at least one `Finding` on a synthetic fixture
- retrieval returns the expected citation shape (mocked MedRAG)
- safety checker rejects hallucinated citation ids
